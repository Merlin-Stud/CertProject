# ТОГУ — Платформа электронных наградных документов

Flask-приложение для формирования, выдачи, хранения и верификации
электронных благодарностей, грамот, дипломов и сертификатов
участникам мероприятий Тихоокеанского государственного университета.

Реализовано строго по ТЗ и в соответствии с Брендбуком ТОГУ 2023.

---

## 📋 Соответствие ТЗ

| Раздел ТЗ | Реализация |
|-----------|------------|
| 1.1 Защищённая веб-платформа | Flask + сессионная авторизация, хэширование паролей (Werkzeug PBKDF2) |
| 1.2 Соответствие брендбуку | Цвета Pantone 7420 C, шрифты Manrope/Russo One, фирменный знак ТОГУ, паттерн, узор |
| 1.3 Автоматизация (импорт, массовая выдача) | CSV-импорт, массовый выпуск через мастер |
| 2.1–2.3 Область применения | 5 типов документов: диплом победителя/участника, благодарность, грамота, сертификат. Форматы A4/A5, PDF/PNG (через preview) |
| 3.1–3.5 Роли | 5 ролей: суперадмин, админ подразделения, проверяющий, получатель, наблюдатель |
| 4.1 Управление мероприятиями | Создание, импорт CSV (ФИО, e-mail, статус, достижения, часы) |
| 4.2 Конструктор шаблонов | С блокировками брендбука (шрифты, паттерн, цвета, охранное поле) |
| 4.3 Генерация документов | Уникальный № `ТОГУ-{год}-{HEX}`, QR-код, КЭП (SHA-256), массовая выдача |
| 4.4 Личный кабинет получателя | Доступ по персональной защищённой ссылке (token), просмотр, скачивание PDF, проверка подписи |
| 4.5 Соблюдение бренда | Палитра, шрифты, логотип, грид-система — на уровне CSS и генерации PDF |
| 4.6 Соответствие 152-ФЗ | Поле `consent_pd` у получателей, журналирование действий |
| 5 Иное | Мастер выпуска, подсказки по бренду, страница проверки (тёмная/светлая тема), модель данных (User, Event, Recipient, Document, Template, EventLog) |

## 🚀 Запуск в PyCharm

### 1. Откройте проект
```
File → Open → выберите папку togu_certs
```

### 2. Создайте виртуальное окружение (если ещё нет)
```
PyCharm: Settings → Project → Python Interpreter → Add → Virtualenv
```
Либо через терминал PyCharm:
```bash
python -m venv .venv
.venv\Scripts\activate     # Windows
source .venv/bin/activate   # macOS/Linux
```

### 3. Установите зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустите приложение
В PyCharm: правый клик на `app.py` → **Run 'app'**

Либо через терминал:
```bash
python app.py
```

### 5. Откройте в браузере
```
http://127.0.0.1:5000
```

## 🧪 Демо-сценарий

1. Войдите как `admin@pnu.edu.ru` / `admin123`
2. **Мероприятия → + Создать мероприятие** — заполните название, тип, даты
3. Откройте созданное мероприятие → **⤓ Импорт CSV** — загрузите файл `sample_recipients.csv` из корня проекта
4. Отметьте получателей чекбоксами, выберите шаблон (например, «Диплом победителя»), нажмите **⚡ Выпустить документы**
5. В таблице «Выпущенные документы» нажмите **PDF** — скачается готовый документ с QR-кодом
6. Нажмите **QR** — откроется публичная страница проверки подлинности
7. Зайдите в **Шаблоны** — попробуйте изменить цвет института (брендбук-блокировки применены)
8. **Журнал** — посмотрите аудит-лог всех действий
9. Выйдите → войдите как `student@pnu.edu.ru` — увидите свои документы (фильтрация по e-mail)

## 🎨 Шрифты для PDF (опционально)

По умолчанию PDF использует встроенный Helvetica с поддержкой кириллицы.
Для **полного соответствия брендбуку** добавьте TTF-файлы:

1. Скачайте [Manrope](https://fonts.google.com/specimen/Manrope) и [Russo One](https://fonts.google.com/specimen/Russo+One)
2. Положите в `static/fonts/`:
   - `Manrope-Regular.ttf`
   - `Manrope-Bold.ttf`
   - `RussoOne-Regular.ttf`
3. Перезапустите приложение — шрифты подхватятся автоматически

## 🔐 Безопасность

- Пароли хэшируются (PBKDF2 + соль)
- Сессионные токены — secrets.token_hex(32)
- Доступ к документам — по уникальному токену (token_urlsafe(24))
- Каждый документ подписан SHA-256(данные + secret_key) — имитация УКЭП
- Журналирование действий (User-IP-action-entity)
- Защита от загрузки больших файлов (16 МБ лимит)
- Декораторы `@login_required` и `@role_required(...)` на каждом маршруте
